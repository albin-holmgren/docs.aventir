## Integration Architecture

Aventir uses a Model Context Protocol (MCP) architecture for integrations:

```
Agent → Integration Runtime → Provider API
```

### How It Works

1. **Agent requests action** - "Send Slack message"
2. **Integration Runtime validates** - Check credentials, rate limits
3. **Execute API call** - Call Slack API with proper authentication
4. **Return result** - Success/failure back to agent

### Authentication Methods

| Method | Description | Best For |
|--------|-------------|----------|
| **OAuth 2.0** | Token-based, user consent | Most services |
| **API Key** | Direct authentication | Backend services |
| **Password** | Username/password legacy | Services without OAuth |

## Adding an Integration

### Step 1: Choose Service

Navigate to **Settings** → **Integrations** and select the service you want to connect.

### Step 2: Authenticate

**For OAuth:**
1. Click **Connect**
2. Complete OAuth flow in popup
3. Grant requested permissions
4. Automatically connected

**For API Key:**
1. Enter API key from service dashboard
2. Select appropriate permissions
3. Test connection
4. Save credentials

### Step 3: Verify Connection

Test the integration:

```
"Send a test message to Slack"
"Check if my Gmail connection works"
```

### Step 4: Use in Agents

Once connected, agents can automatically use the integration:

```
"Check my Slack messages every hour"
"Post Shopify orders to our Slack channel"
```

### Disconnecting Services

To remove an integration:

1. Go to **Settings** → **Integrations**
2. Find the service
3. Click **Disconnect**
4. Confirm removal

<Warning>
Disconnecting a service will cause agents using it to fail. Update or pause those agents first.
</Warning>

### Reconnecting

If authentication expires:

1. Integration shows "Attention Required"
2. Click **Reconnect**
3. Complete OAuth flow again
4. Connection restored

## Integration Best Practices

### Use OAuth When Available

OAuth is more secure than API keys:

- Tokens can be revoked
- Scoped permissions
- No password exposure
- Automatic refresh

### Minimal Permissions

Grant only needed permissions:

**Good:**
```
Slack: chat:write in #alerts only
Gmail: read-only inbox access
```

**Avoid:**
```
Slack: Full workspace admin
Gmail: Send emails as any user
```

### Dedicated Accounts

For business automation:

- Create service accounts (e.g., `automation@company.com`)
- Separate from personal accounts
- Easier permission management
- Clear audit trail

### Monitor Usage

Track integration activity:

```
Gmail API Usage (Last 30 Days):
- API calls: 450
- Rate limit: 1,000/day
- Status: Healthy
```

### Regular Audits

Review connected services monthly:

- Remove unused integrations
- Update expired credentials
- Verify permissions still appropriate
- Check for new scopes available

## Troubleshooting

### Connection Failed

**Symptoms:** Unable to connect service

**Solutions:**
- Check popup blocker settings
- Verify third-party cookies enabled
- Ensure you have admin access to service
- Try incognito/private window

### Authentication Expired

**Symptoms:** Integration shows "Token expired"

**Solutions:**
- Click **Reconnect**
- Re-authorize the app
- Check if account password changed
- Verify app not revoked in service dashboard

### Rate Limit Errors

**Symptoms:** Agent fails with "rate limit exceeded"

**Solutions:**
- Reduce agent frequency
- Check service rate limits
- Upgrade service plan if needed
- Use multiple accounts (if supported)

### Permission Denied

**Symptoms:** Agent can't perform action

**Solutions:**
- Check granted permissions in service dashboard
- Reconnect with broader scopes
- Verify account has required access level
- Contact service admin

### Request New Integration

Need a service not listed?

2. Describe the service and use case

### API-Based Integration

For services without native integration:

```bash
# Use the Aventir API to build custom workflows
curl -X POST https://api.aventir.io/v1/agents \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "taskDescription": "Call custom API endpoint..."
  }'
```

## Security

### Data Handling

- Credentials encrypted with AES-256-GCM
- API calls made server-side (never exposed to client)
- Tokens refreshed automatically
- Access logged for audit

### Compliance

- OAuth 2.0 industry standard
- No password storage for OAuth services
- Scoped permissions minimize exposure
- Compliance with service ToS
